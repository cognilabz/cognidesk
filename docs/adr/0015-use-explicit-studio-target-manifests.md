# Use explicit Studio Target manifests

Cognidesk Studio will bind to SDK applications through versioned, non-executable Studio Target Manifests rather than implicit environment discovery or target-owned TypeScript config execution. Target repositories may provide non-secret manifest defaults in `cognidesk.studio.json`, while the Studio deployment supplies environment-specific overlays; Studio merges them deterministically into an effective manifest visible in Admin. This makes the runtime API, observability source, source repository, allowed code-change scope, dashboard storage, credential grant categories, and permission boundaries reviewable before the Studio Operator can inspect code, generate dashboards, or prepare pull requests; secrets and environment-specific endpoint values still belong outside repo defaults.
