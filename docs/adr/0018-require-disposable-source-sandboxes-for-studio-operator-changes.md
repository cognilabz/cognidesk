# Require disposable source sandboxes for Studio Operator changes

The Studio Operator will inspect and modify target source code only through disposable Studio Source Sandboxes, constrained by the Studio Target Manifest's allowlisted write scope, rather than mutating a live mounted repository. Studio Source Sandboxes have full outbound network access so validation can install dependencies, reach registries, inspect external documentation, and interact with GitHub realistically; when a goal needs credentials for backend services, those credentials are provided through explicit audited Studio Credential Grants. This keeps generated edits reviewable, prevents accidental runtime hot patches, and makes GitHub pull requests the boundary where proposed source changes become durable while putting containment, credential grants, and audit logging on the sandbox boundary.
