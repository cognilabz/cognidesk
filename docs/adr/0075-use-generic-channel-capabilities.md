# Use generic channel capabilities

Cognidesk will model features such as receiving, sending, drafting, threading, attachments, media, handoff, scheduling, cleanup, Provider Object operations, notifications, transfers, and artifacts as generic Channel Capabilities rather than as email-specific, voice-specific, or ticketing-specific business concepts. Customer-facing, internal support, and mixed visibility are Action Audience metadata rather than separate capabilities. Cognidesk will provide a core capability vocabulary while allowing Provider Integrations and SDK users to declare Extension Channel Capabilities, which must be visible in Studio and policy-governed when they affect external state, sensitive data, workflow, Outbound Contact, Support Artifacts, or customer-visible behavior.
