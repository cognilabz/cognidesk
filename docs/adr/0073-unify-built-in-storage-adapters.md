# Unify built-in storage adapters in one package

Cognidesk will publish built-in Node storage adapters from a single storage library, with provider-specific subpath exports such as `@cognidesk/storage/sqlite` and `@cognidesk/storage/postgres`, rather than keeping one package per storage backend. This supersedes ADR-0005's separate SQLite adapter package direction: the storage contract remains `StorageAdapter` in Core, while the unified storage package improves discoverability and reduces package sprawl. Built-in adapters will run non-destructive schema migrations through an idempotent storage initialization lifecycle, SQLite and Postgres will both use Drizzle for schema and type-safe database access, SQLite will use the `@libsql/client` driver, SQL adapters will accept either simple connection options or application-managed clients, provider factories will live only on provider subpaths so the root import stays lightweight, `@cognidesk/storage/conformance` will expose a reusable conformance suite for all Storage Adapter implementations, SQL adapters will share one logical schema while allowing dialect-specific physical types, storage will index runtime envelope fields including telemetry trace/span identifiers while treating event and snapshot JSON payloads as opaque, `listEvents` will continue returning full history when no limit is provided, conformance will require safe concurrent event appends and reject orphan Runtime Events or Runtime Snapshots while keeping missing lifecycle updates as `null`, no public in-memory adapter will be shipped, and the existing SQLite-only package will be removed rather than kept as a compatibility wrapper.
